# 访客设备识别与封禁能力

## 用户故事

- 作为管理员，我希望在访客日志里看见可识别的设备信息，这样我能知道是谁在刷题、谁在重复访问。
- 作为管理员，我希望能一键封禁某个设备，这样可疑访问可以立刻被拦住。
- 作为管理员，我希望能按 IP 封禁或解封，这样在设备标识失效时还能有备用手段。
- 作为管理员，我希望能看到每条封禁记录的原因，这样后续排查时能知道为什么封。

## 为什么需要

当前访客日志只提供 visitorId、IP、来源地和访问次数，能看见“发生了什么”，但不能直接处理“这个访客该不该继续访问”。当发现恶意访问、异常刷题、测试流量污染日志时，管理员需要一种能立即生效的封禁手段。

## 怎么解决

把浏览器里可稳定获取的匿名访客标识作为设备 ID，用它和 IP 作为两类封禁对象。管理员在访客日志页就能看到设备信息、封禁状态和封禁按钮，并在封禁时填写原因。

## 边界

- 不负责真实 MAC 地址采集；标准浏览器 Web 端拿不到这个值。
- 不负责把所有请求都做成强身份认证。
- 不负责取代访客日志本身的统计能力。
- 不负责跨设备的账号级封禁；这次是访客级和 IP 级封禁。

## 验收标准

- 访客日志可以看到设备信息和封禁状态。
- 管理员可以对设备执行封禁和解封。
- 管理员可以对 IP 执行封禁和解封。
- 封禁可以写入原因。
- 被封禁的访客在 heartbeat 时会收到 403。
