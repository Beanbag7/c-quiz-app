# 填空题连续作答体验

## 用户故事

- 作为一个连续刷题的人，我希望填空题按回车就能提交，这样不用每题都去点按钮。
- 作为一个答对题目的用户，我希望系统自动跳到下一题，这样我可以不停地往下做。
- 作为一个答错题目的用户，我希望系统先停一下让我看清反馈，再由我手动继续，这样不会错过正确答案。
- 作为一个连续输入的人，我希望下一题打开后输入框自动聚焦，这样我可以直接接着打字，不用再点一次输入框。

## 为什么需要

填空题是高频连续输入场景，最怕“提交一次、点一次、再点一次”。如果提交、判题、切题、聚焦之间没有统一约定，用户就会在连续作答时不断被打断，体验会明显比单选题差。

## 怎么解决

把填空题做成一条连续作答闭环：输入后直接回车提交；答对后自动进入下一题；答错后保留手动确认；题目切换后输入框自动聚焦。

## 边界

- 不负责单选、多选、判断、解答题的连续作答逻辑。
- 不负责题库数据本身的 schema 清洗。
- 不负责错误答案自动跳题；错误答案必须由用户手动继续。
- 不负责浏览器自动化工具本身的键盘兼容问题，但验收时要把这个问题纳入检查。

## 验收标准

- 填空题输入框按回车可提交。
- 答对后会自动进入下一题。
- 答错后不会自动跳题。
- 进入下一题后输入框会自动获得焦点。
- 自动切题不会因为退出/切题导致串题。
