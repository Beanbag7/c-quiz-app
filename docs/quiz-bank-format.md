# 规范化题库数据标准

> 本文件是 c-quiz-app 题库 JSON 的权威格式规范。新增、清洗、导入、维护题库时必须以此为准。

## 顶层结构

```json
{
  "meta": {
    "subjectKey": "co",
    "name": "计算机组成与系统结构",
    "version": "2026-06-22"
  },
  "questions": [ ... ]
}
```

`meta` 推荐填写；`questions` 必填，必须是数组。

## 必填字段（每道题）

| 字段 | 说明 |
|------|------|
| `序号` | 整数，题号，在同一题库内稳定 |
| `题目类型` | 见下方支持的类型 |
| `题目内容` | 题干文本，支持 Markdown |
| `正确答案` | 格式随题型变化（见下方） |
| `答案文本` | 用于展示的答案内容，支持 Markdown |

## 推荐字段

| 字段 | 说明 |
|------|------|
| `题目ID` | 唯一标识 |
| `章节` | 章节归属 |
| `选项` | 客观题必填，**必须是对象不是数组** |
| `解析` | 题目解析，支持 Markdown，可填 `null` |
| `分值` | 数值 |
| `必考` | 布尔值 |
| `题目图片` | 题干图片数组，路径以 `/images/` 或 `/assets/` 开头 |

## 支持的题型

- `选择题`（新建题库优先使用）
- `单选题`（兼容旧题库；新题库建议写 `选择题`）
- `多选题`
- `判断题`
- `填空题`
- `解答题`
- `模拟卷`

## 题型适配矩阵

| 题型 | `选项` | `正确答案` 标准类型 | `答案文本` | Markdown 字段 | 通用练习能力 |
|------|--------|---------------------|------------|---------------|--------------|
| `选择题` / `单选题` | 必填，对象 | 字符串，如 `"A"` | 选项解释或答案全文 | 题干、选项、答案、解析 | 查看答案、跳过、下一题 |
| `多选题` | 必填，对象 | 字符串数组，如 `["A","C"]` | 答案全文 | 题干、选项、答案、解析 | 查看答案、跳过、下一题 |
| `判断题` | 推荐填 `{ "A": "正确", "B": "错误" }` | 字符串，如 `"A"` | 正确/错误或解释 | 题干、选项、答案、解析 | 查看答案、跳过、下一题 |
| `填空题` 单空 | 不填 | 字符串或可接受答案数组 | 展示答案 | 题干、答案、解析 | 多输入框、查看答案后仍可练习、跳过 |
| `填空题` 多空 | 不填 | 按空位顺序的数组 | 展示答案 | 题干、答案、解析 | 每空一个输入框、查看答案后仍可练习、跳过 |
| `解答题` | 不填 | 字符串，通常写 `"见答案文本"` | 完整参考答案 | 题干、答案、解析 | 查看答案后仍可输入练习、跳过 |
| `模拟卷` | 不填 | 字符串，通常写 `"见答案文本"` | 整卷参考答案 | 题干、答案、解析 | 按整套试卷练习、查看答案后仍可输入练习、跳过 |

所有题型都应能显示 Markdown 表格、列表、代码块、行内代码和图片。题目涉及代码时，优先使用围栏代码块，避免把代码挤在一行纯文本中。

## Markdown 富文本规范

以下字段统一支持 Markdown：

| 字段 | 支持范围 |
|------|----------|
| `题目内容` | 段落、列表、表格、行内代码、代码块、粗体、图片 |
| `选项.A/B/C...` | 段落、行内代码、图片、简单表格 |
| `答案文本` | 段落、列表、表格、行内代码、代码块、粗体、图片 |
| `解析` | 段落、列表、表格、行内代码、代码块、粗体、图片 |

Markdown 表格必须使用标准写法：

```md
| 方式 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| 集中刷新 | 在固定时间段内刷新所有行 | 控制简单 | 存在死区 |
| 分散刷新 | 每个周期轮流刷新各行 | 无死区 | 速度下降 |
```

代码块必须使用围栏代码块：

````md
```c
typedef struct BTNode {
  TElemType data;
  struct BTNode *lchild,*rchild;
} BTNode;
```
````

图片使用 Markdown 图片语法：

```md
![答案图片](/images/ds_essay/66_1.png)
```

禁止事项：

- 不要把 Markdown 表格写成普通段落后再手工对齐
- 不要在 JSON 字符串中直接写未转义的换行，必须使用 `\n`
- 不要混用不完整 HTML 标签；图片优先用 Markdown 语法
- 不要把题干图片混入 `题目内容`，题干主图优先放 `题目图片`

## 各题型示例

### 1. 选择题

```json
{
  "序号": 1,
  "题目ID": 100001,
  "章节": "第1章",
  "题目类型": "选择题",
  "题目内容": "关于信息，以下说法正确的是（）",
  "选项": {
    "A": "信息 = 数据 + 语义",
    "B": "信息 = 数据",
    "C": "信息 = 数据 - 语义",
    "D": "信息 = 语义"
  },
  "正确答案": "A",
  "答案文本": "信息 = 数据 + 语义",
  "解析": "数据是信息的符号表示。",
  "分值": 1,
  "必考": false
}
```

`正确答案` 是**单个字符串**。

客观题选项也可以使用 Markdown，例如：

```json
{
  "选项": {
    "A": "`int *p` 表示指针变量",
    "B": "![选项图](/images/options/a.png)"
  }
}
```

### 2. 多选题

```json
{
  "序号": 2,
  "题目ID": 100002,
  "章节": "第1章",
  "题目类型": "多选题",
  "题目内容": "下列哪些说法正确？",
  "选项": {
    "A": "选项A",
    "B": "选项B",
    "C": "选项C",
    "D": "选项D"
  },
  "正确答案": ["A", "C"],
  "答案文本": "选项A、选项C",
  "解析": null,
  "分值": 1,
  "必考": false
}
```

`正确答案` 是**字符串数组**。

### 3. 判断题

```json
{
  "序号": 3,
  "题目ID": 100003,
  "章节": "第1章",
  "题目类型": "判断题",
  "题目内容": "高质量发展是首要任务。",
  "选项": {
    "A": "正确",
    "B": "错误"
  },
  "正确答案": "A",
  "答案文本": "正确",
  "解析": null,
  "分值": 1,
  "必考": false
}
```

### 4. 填空题（单空）

```json
{
  "序号": 4,
  "题目ID": 100004,
  "章节": "第1章",
  "题目类型": "填空题",
  "题目内容": "中国式现代化的本质要求之一是 ______ 。",
  "正确答案": "实现高质量发展",
  "答案文本": "实现高质量发展",
  "解析": null,
  "分值": 1,
  "必考": false
}
```

单空多个可接受答案时：

```json
{
  "正确答案": ["实现高质量发展", "推动高质量发展"],
  "答案文本": "实现高质量发展"
}
```

### 5. 填空题（多空）

多空题必须按空位顺序写数组，前端会自动生成多个输入框：

```json
{
  "序号": 5,
  "题目ID": 100005,
  "章节": "第2章",
  "题目类型": "填空题",
  "题目内容": "数据结构包括数据的（ ）、数据的（ ）和数据的（ ）。",
  "正确答案": ["逻辑结构", "存储结构", "运算"],
  "答案文本": "逻辑结构、存储结构、运算",
  "解析": null,
  "分值": 1
}
```

规则：
- `正确答案` 数组长度应等于题干空位数
- 多空按顺序判分，不做乱序匹配
- 空位建议写成 `（ ）` 或 `( )`

### 6. 解答题

```json
{
  "序号": 5,
  "题目ID": 100006,
  "章节": "第1章",
  "题目类型": "解答题",
  "题目内容": "简述 DRAM 为什么需要刷新，并比较三种刷新方式。",
  "正确答案": "见答案文本",
  "答案文本": "(1) DRAM 靠电容存储电荷，电荷会泄漏，因此必须周期性刷新。\n\n| 方式 | 做法 | 优点 | 缺点 |\n|------|------|------|------|\n| 集中刷新 | 集中刷新所有行 | 控制简单 | 存在死区 |\n| 分散刷新 | 轮流刷新各行 | 无死区 | 周期变长 |\n| 异步刷新 | 分散安排刷新 | 效率较高 | 控制复杂 |",
  "解析": null,
  "分值": 5,
  "必考": false
}
```

#### 解答题答案图片（Markdown 语法）

解答题的参考答案常常包含图片（结构图、过程图、手写答案截图等）。
答案里的图片用 **Markdown 图片语法**内嵌到 `答案文本` 中，与文字自然混排：

```json
{
  "答案文本": "逻辑结构指数据元素间抽象的相互关系。四类如下：\n\n![答案图片](/images/ds_essay/66_1.png)\n\n集合、线性结构、树形结构、图形结构。",
  "题目图片": ["/images/ds_essay/66_q_1.png"]
}
```

规则：
- 图片路径是 **相对 public 根目录** 的绝对路径，以 `/images/` 开头
- 语法为 `![替代文字](路径)`，前端会渲染为 `<img>`
- 文字和图片可在 `答案文本` 中任意穿插，用空行分隔段落
- 题干图片写入独立字段 `题目图片`（字符串数组），不混入 `题目内容`
- 纯文本答案（无图片）保持原样即可

图片来源：图片文件存放在 `public/images/<题库标识>/` 下，由抓取脚本从超星下载（带 Referer 防盗链）。

## 字段规范化规则

导入前应将源数据统一清洗为以下标准：

| 源字段/旧写法 | 标准字段 |
|---------------|----------|
| `题干`、`question`、`content` | `题目内容` |
| `答案`、`answer` | `正确答案` |
| `答案解析`、`analysis` | `解析` |
| `单选题` | 可保留兼容；新增建议改为 `选择题` |
| `"A,C"` 多选答案 | `["A", "C"]` |
| 选项数组 `["a","b"]` | 选项对象 `{ "A": "a", "B": "b" }` |

标准化后必须保证：

- 字段名使用中文标准字段
- `题目类型` 只能使用支持列表中的值
- 客观题 `选项` key 使用大写字母 `A/B/C/D...`
- 客观题 `正确答案` 写选项 key，不写选项全文
- 解答题 `答案文本` 是最终展示给用户的完整参考答案
- 需要排版的答案优先用 Markdown，不使用临时空格对齐

## 关键规则（容易踩坑）

1. **`选项` 必须是对象** `{ "A": "...", "B": "..." }`，不是数组 `["...", "..."]`
2. **多选题答案必须是数组** `["A", "C"]`，不能是逗号字符串 `"A,C"`，不能写选项内容
3. **选择题/判断题答案是单个字符串** `"A"`
4. **字段名固定**，不能用别名：
   - ❌ `题干` → ✅ `题目内容`
   - ❌ `答案` → ✅ `正确答案`
   - ❌ `说明` → ✅ `解析`
5. **`答案文本` 必须填**，即使源数据只有答案 key，也要补上实际内容
6. **填空题/解答题不需要 `选项` 字段**
7. **Markdown 表格要写在 `答案文本`/`解析`/`题目内容` 中**，不要截图替代可编辑文本
8. **JSON 字符串里的换行必须写成 `\n`**

## 快速校验清单

- [ ] 顶层是 `{ "questions": [...] }`
- [ ] 每道题都有 `题目类型`、`题目内容`、`正确答案`、`答案文本`
- [ ] 客观题 `选项` 是对象，不是数组
- [ ] 多选题 `正确答案` 是字符串数组
- [ ] 选择题/判断题 `正确答案` 是单个字符串
- [ ] `答案文本` 与正确答案内容一致
- [ ] 题干、选项、答案、解析中的 Markdown 表格可正常解析
- [ ] 图片路径以 `/images/` 或 `/assets/` 开头
- [ ] 输出是合法 JSON，没有多余文本
